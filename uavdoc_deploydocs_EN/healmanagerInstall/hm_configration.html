Configuration

Configuration

HM、AppHub、MA的配置都分为本地环境配置、测试环境配置(test字样)和生产环境配置(pro字样)，同一环境的配置文件端口信息相互对应，部署时若修改了HM服务的端口，则MA、AppHub相应的端口也需修改。

HM configuration

The configuration file of HM is under the directory uavhm/config. Each feature of resource is with independent configuration snippet.

Common configuration items

meta.nodetype
Name of service node

resource.*.class
Component class of resources (please refer to agent.properties)

feature.*.loader
Class loader of features. Fill in the name of the jar package where the feature belongs to (please refer to agent.properties). Fill in “Default” to use the default class loader.

feature.*.enable
Enable this feature or not

feature.*.components
Component classes that compose feature *.http.enable
Enable this Http service or not

*.XXXHandlers
Handler class of Http request. Separate multiple classed with ‘,’

*.http.port
Port of Http service

*.min(core)/max/bqsize
core thread count in the thread pool, maximum thread count, size of the buffer queue

Basic feature/resource

messageproducer

Support MQ message producer

resource.messageproducer.messagingnameserver

rocketmq Nameserver address. Separate multiple addresses with “;”

resource.messageproducer.messagingthreshold

Compression threshold, in kb. Compress before delivery if the message size is above the threshold

httpinvoke

Support service discovery invocation

resource.httpinvoke.http.client.maxperroute

Maximum per route connection of Http invocation

resource.httpinvoke.http.client.maxtotal

Maximum connection of Http invocation

resource.httpinvoke.http.client.sotimeout

Socket connection timeout

resource.httpinvoke.http.client.conntimeout

Http connection timeout

resource.httpinvoke.http.client.reqtimeout

Http request timeout

nodeoperagent

Support node control

feature.nodeoperagent.http.port=10140

Receiving port of node control messages

notifyagent

Support notification

feature.notifyagent.notifyhandlers

Handler class of the notification

feature.notifyagent.expireTime

Expiration time of notification event, in s

feature.notifyagent.frozenTime

Frozen time of notification messages in s, indicating that the same notification shall not be repeatedly responded within this time span

hbclientagent

Support the sending of the hearbeat information of nodes

feature.hbclientagent.hbhandlers
Handler class of the heartbeat information

feature.hbclientagent.hbservers
Address of heartbeat service. Separate multiple addressed with ‘,’

feature.hbclientagent.interval
Heartbeat sending interval (in ms)

Functional feature

healthmanager

Support the receiving and processing of the monitoring data and the profile data, the storage of the heartbeat data, and the receiving and forwarding of the notification data

feature.*.consumethreadmax
Maximum consumer thread count of the message

feature.*.consumestopinterval
Stop interval of message consumption

feature.*.enable
Consume the meaasge of this topic or not

feature.healthmanager.*.ds.enable
Store this type of message or not

feature.healthmanager.*.ds.servers
Message storage address Store the monitoring data(MT_Monitor) and the container node data(MT_Node) in OpenTSDB; store the profile data(MT_Profile) and the notification data(MT_Notify) in mongodb

feature.healthmanager.*.ds.db/usr/pwd
Datastore name / Username / password (if any).

feature.healthmanager.*.ds.retry
Number of retries of the OpenTSDB insert request

feature.healthmanager.lifekeeper.interval
Expiration time of the cache of the profile data

feature.healthmanager.store.addr
Address of profile data cache (redis)

notifycenter

Support notification information processing and sending

feature.notifycenter.nc.notify.ttl
Expiration time of the first notification, in hour

feature.notifycenter.nc.notify.frozenTime
Time interval of sending the same notification, in s

feature.notifycenter.nc.cache.addr
Address of notification cache (Redis)

feature.notifycenter.push.services
Push Http service address in the notification (optional)

feature.notifycenter.nc.notify.mail.provider
Email delivery mode (CEMail within CreditEase. JavaMail for others)

feature.notifycenter.nc.notify.mail.msgurl
Access address of notification details (Access address of AppHub /uavapp_godeye/notifymgr/desc.html)

feature.notifycenter.nc.notify.mail.uavurl
AppHub access address (such as http://127.0.0.1:8080/apphub)

feature.notifycenter.nc.notify.mail.javamail.smtphost
Smtp service address (such as mail.163.com/)

feature.notifycenter.nc.notify.mail.javamail.smtpport
Port of Smtp service

feature.notifycenter.nc.notify.mail.javamail.smtpauth
Validation or not: true

feature.notifycenter.nc.notify.mail.javamail.username
Outbox address (such as XXX@163.com)

feature.notifycenter.nc.notify.mail.javamail.password
Outbox password

hbserveragent

Support heartbeat data processing

feature.hbserveragent.http.port
Port of the heartbeat data receiving service

feature.hbserveragent.http.qport
Port of the heartbeat data query service

feature.hbserveragent.store.addr
Address of heartbeat data cache (Redis)

feature.hbserveragent.ismaster
Endpoint of heartbeat or not. If yes, process the heartbeat data; if no, send to the next heartbeat

feature.hbserveragent.lifekeeper.interval
Heartbeat interval (in ms)

feature.hbserveragent.lifekeeper.locktimeout
Expiration time of the cache lock of heartbeat data, in ms

feature.hbserveragent.lifekeeper.nodetimeout
Expiration time of heartbeat, in ms A node shall be assumed as probably having stopped working if no heartbeat data of it has been received before the expiration time and as having stopped working if no data has been received five times late than the expiration time. The node shall be removed if no data has been received ten times late than the expiration time.

apphubmanager

Support interaction with AppHub data

feature.apphubmanager.ds.servers

Address of AppHub management data storage (Mongo)

runtimenotify

Support runtime notification decision-making

feature.runtimenotify.MT_Runtime.enable<br Consume the source data of runtime notification or not

feature.runtimenotify.storecm.server
Address of the sliced cache of the runtime notification source data (Redis)

feature.runtimenotify.nodeinfotimer.enable
Capture the heartbeat data information or not

feature.runtimenotify.nodeinfotimer.period
Interval of heartbeat data capturing

feature.runtimenotify.nodeinfoprocess.sendmq
Send the heartbeat data to MQ or not

feature.runtimenotify.nodeinfoprocess.exchange
Directly use the heartbeat data for notification decision-making or not

feature.runtimenotify.crash.timeout
Expiration time of process information, in ms Determine that the process has stopped if no information about the process is detected before the expiration time

feature.runtimenotify.strategy.interval
Time interval of notification strategy update, in ms

collectserver

Support APM data (invocation chain and thread analysis) receiving

feature.collectserver.topics
Topic type of the subscried messages. Separate multiple types with “,”

newlogservice

Support log data processing

invokechainserver

Support invocation chain data processing

threadanalysisserver

Support thread analysis

feature.*.es.addr
Data storage (elasticsearch) address

feature.*.es.clustername
Elasticsearch cluster name

feature.*.es.query.timeout
Timeout of Elasticsearch query, in ms
