Invocation Chain Data Collection

Invocation Chain Data Generation

The invocation chain is the specific implementation of the domain model of distributed tracing. The invocation chain records each request and analyzes the invocation process.

Invocation chain model

  • Invocation chain metadata: SpanEndpointType: invocation type (Root(“E”), Service(“S”), Client(“C”), Method(“M”)); traceId: unique identifier of trace; spanId: invocation sequence of the current node in an invocation chain (unique when combined with SpanEndpointType) parentId: parent invocation node of current node in an invocation chain.

  • Invocation chain mapping rules: 1) Register the initial invocation (without parent invocation) of the invoker (service and web) as the starting node E and generate the unique traceID; 2) Add “1” (“.1” if comes as the first one) after the spanID for the invocation among application components (such as httpclient and method invocation) within the system; 3) Span information of Service A and Service B differs in SpanEndpointType only (each representing one end of the span) for the invocation among systems.

  • Illustration with the example:

  • traceId;1;N;E
  • traceId;1.1;1;C
  • traceId;1.1;1;S
  • traceId;1.1.1;1.1;C
  • traceId;1.1.2;1.1;C

Invocation chain model implementation

  • The invocation chain employs the GlobalFilter mechanism of UAV to implement JEEServiceRunGlobalFilterHandler and intercepts all requests to access Tomcat.
  • Abstract three scenes: 1, server; 2, client; and 3, method. Create corresponding handlers.
  • Add adapters (XXAdapter) to the hook of the client so as to adapt all data sources (such as HttpClient, MQ and JDBC).
  • Output the invocation chain information to the log file with the mechanism aforesaid and collects logs with the collection mechanism of MA.

Invocation chain architecture

1) Enable or disenable the invocation chain with the inline applets of UAV;

2) MOF collects the invocation chain information and outputs it to the log files;

3) The collection mechanism of MA uploads logs to the specifized topic in RocketMQ of UAV;

4) The invocation chain handlers in HM analyze and process the invocation chain data in RocketMQ and store the data in ElasticSearch;

5) HM supports the query interfaces for front-end display of the console.
